# Onno 개발 마스터 가이드

**버전**: 2.1
**최종 수정**: 2025-12-30
**목적**: 모든 기획 문서를 통합 참조하여 누락 없이 개발하기 위한 마스터 가이드
**사용법**: Part별로 개발 시 이 문서를 먼저 확인 → 상세 내용은 각 문서 참조

## 🚀 개발 방법론: 단계적 고도화 (Incremental Development)

### 핵심 원칙

```
┌─────────────────────────────────────────────────────────────────────┐
│                     Onno 개발 철학                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  1. 복사 금지, 참조만 허용                                           │
│     → 기획 내용을 복사하면 누락 발생                                  │
│     → 원본 문서의 라인 번호만 참조                                    │
│     → `00_개발_참조맵.md` 활용                                       │
│                                                                     │
│  2. Part 단위 완결성                                                 │
│     → 각 Part는 독립적으로 동작해야 함                                │
│     → 횡단 관심사(6.1-6.4)는 모든 Part에 적용                         │
│                                                                     │
│  3. 문서 ↔ 코드 동기화                                               │
│     → 코드 변경 시 이 문서의 진행 상태 업데이트                        │
│     → 기획 변경 시 원본 문서 수정 (절대 마스터 가이드에 복사 X)         │
│                                                                     │
│  4. 테스트 우선                                                      │
│     → 각 Part 완료 시 해당 Part의 E2E 테스트 작성                     │
│     → 횡단 관심사 테스트 별도 구성                                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

━━━━━━━━━━━━━━━━━━━━━

━━━━━━━━━━━━━━━━━━━━━
